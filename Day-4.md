## Day-4 Checklist

- [x] I know the 5 core HTTP methods (GET, POST, PUT, PATCH, DELETE) and what each is used for
- [x] I can read a status code and know what went wrong — e.g., 401 vs. 403 vs. 404 vs. 500
- [x] I can open Chrome DevTools Network tab, find a request, and inspect its headers, payload, and response
- [x] I can copy a browser request as a cURL command and run it in the terminal
- [x] I can change the `User-Agent` in the browser and see the change in the Network tab
- [x] I can use `curl` to make a GET request with query parameters and a POST request with a JSON body
- [x] I have a running FastAPI app with at least two endpoints (`GET /health` and `POST /echo`)
- [x] I can test my API using the Swagger UI at `/docs` and via `curl` from the terminal

--- After Day-4 ---
I reinforced my understanding of how web applications communicate over HTTP and gained more confidence in debugging API requests using browser developer tools. I also practiced inspecting request headers, payloads, and responses, which helped me better understand client-server communication. Revisiting cURL commands and testing APIs through both Swagger UI and the terminal strengthened my workflow for backend development and API debugging.

--- Feedback (Suggestions for the TDS Team) ---
The hands-on approach was very useful. Since some learners may already have prior experience with FastAPI or backend development, it could be helpful to include optional advanced exercises involving authentication, API rate limiting, request validation, or integrating external APIs. These additions would provide extra challenges for experienced participants while keeping the core material accessible to beginners.
